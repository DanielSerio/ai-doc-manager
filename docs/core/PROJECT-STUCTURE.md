# Project Structure

```
docs/                  # Primary Documentation.
├── core/              # Core Documentation.
├── archive/           # Archived Documentation.
├── specs/             # Gherkin Specifications for particular features. ()
├── implementations/   # Implmentation details for particular features. Contains things like code examples, etc. (`docs/implementations/[FEATURE_NAME].md`)


backend/         # Backend Source Code Folder
web/             # Frontend Source Code Folder

```

## Docs

For details on the documentation workflow, see the [DOCUMENTATION-WORKFLOW.md](DOCUMENTATION-WORKFLOW.md) file.

### Core

Core documentation for the project. This is the global or project-wide documentation. This is where you would put things like development principles, project structure, etc.

### Specs

Specifications for particular features. 

### Tasks

Tasks for particular features.

### Implementations

Implementations for particular features. 

### Archive

Archive. For archiving feature documentation.
